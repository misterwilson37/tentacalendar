⚙️ Settings

Tiers (priority queue)

Rank 1 sorts first. Setting a tier to "Calendar" reveals a Calendar ID field — that tier then holds appointments pulled from that Google Calendar. The day toggles are the tier's working days: reschedules land on them, project dates outside them get flagged, and pipeline offsets only count them (weekend jobs → check Sa/Su). Hover anything in a row for an explanation.

Project stage template

The universal pipeline copied into every NEW project (existing projects keep their own copy). No date = the stage is pipeline weight — it just has to happen before the next dated thing. Before/After = a real target date, offset in working days from project start or end. A stage never surfaces before its predecessors are checked, whatever its date.

Timing

Stages dated by pipeline math (not hard ⏰ dues) are "due by this hour" on their day — so a 9 AM task outranks them, and midnight doesn't flip them red.
